# Border Radius

Our `--radius-base` value translates to `2px` at initial browser scaling and
should be the default choice whenever rounding corners in Jobber. We maintain
this radius for corners on almost all of our components.
[Button](/components/button), [Card](/components/Card),
[InputText](/components/input-text), and [Menu](/components/menu) are a few
examples of this in practice.

Jobber's [marketing site](https://getjobber.com/) uses "sharp" corners with no
rounding as a brand differentiation tactic. This is suitable for a web property
that does not require heavy repeated use, but we use a slight radius on corners
in our software based on research indicating that cognitive and visual strain is
[reduced by rounding corners](/#References). This allows us to maintain a
reasonably smooth transition from the marketing-to-software journey while
optimizing the visual design for repetitive usage.

Larger rounded corners should be used sparingly, but are available for cases
where the user may expect a smoother corner, such as
[ProgressBar](/components/progress-bar). `--radius-circle` can be used for
circular elements like [Radio](/components/radio-group) options or the "pip" in
[Switch](/components/switch).

export function Example({ of: radius }) {
  const style = {
    width: "var(--space-large)",
    height: "var(--space-large)",
    backgroundColor: "var(--color-indigo)",
    borderRadius: `var(--radius-${radius})`,
  };
  return <div style={style} />;
}

| :---------------- | :---------------------- |

| `--radius-larger` | <Example of="larger" /> |
